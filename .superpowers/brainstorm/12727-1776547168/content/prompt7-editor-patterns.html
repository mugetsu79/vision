Sites & Cameras Editing Pattern

Prompt 7 requires create and edit dialogs. The main choice is how heavy those dialogs should feel, especially for the camera form with homography, privacy, and model selection.

A. Center Modal
Sites / cameras table in background
Create camera
Name / processing mode / RTSP
Primary / secondary model / tracker
Homography editor

Center Modal

Familiar and compact. Best if create/edit stays short and focused, but it can feel tight for the camera workflow.

B. Right Drawer
Sites / cameras table remains visible
Edit camera
General fields
Privacy toggles
Homography snapshot
Actions

Right Drawer

Keeps context visible while giving the form more height and a steadier workflow. Strong fit for table-driven CRUD.

C. Split Dialog
Form
Identity + processing
Models + tracker
Privacy
Calibration
Homography editor panel

Split Dialog

More premium and task-focused. Best if the camera form is treated like a serious configuration workflow rather than a quick edit.

My recommendation

  • B for Sites and most Camera edits.
  • C only if you want the homography workflow to feel like a dedicated setup experience from day one.

Watch-out

  • A is simplest but most cramped once we include calibration.
  • C is strongest visually, but it costs more implementation time during Prompt 7.
