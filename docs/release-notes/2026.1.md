# Vezor 2026.1 Release Notes

Date: 2026-06-10
Release type: first product-mode pilot release

Vezor 2026.1 is the first release that brings the core OmniSight workflow
together as an installed product path: first-run identity, tenant operations,
platform superadmin management, master and Jetson edge pairing, model lifecycle,
TensorRT runtime artifacts, live scenes, evidence, deployment posture, and Core
Link diagnostics.

## Highlights

- Live Intelligence workspace with scene selection, live signals, and
  selection-aware Ask Vezor intent.
- Guided scene setup for source, model, privacy, boundaries, and calibration.
- Tenant user management from the UI.
- Platform superadmin bootstrap and platform sign-in for cross-tenant tenant and
  user management.
- Deployment workbench for install health, central and Jetson pairing,
  credential rotation, service reports, and support bundles.
- Model Management workspace for bundled catalog registration, imports, runtime
  artifact builds, TensorRT engines, and edge distribution.
- Jetson edge installer with paired supervisor credential storage, local
  MediaMTX, NATS leaf config, and packaged edge-agent service file.
- Jetson GPU ONNX Runtime wheel resolution from manifest and preflight data for
  dev-manifest edge builds, with manual wheel URL retained as an override.
- Core Link Performance workspace with edge sites, master target posture,
  edge-agent samples, UDP sequence reflector support, and manual throughput
  posture.
- Operations workspace with exception-first readiness, worker status, stream
  diagnostics, deployment node context, and installer guidance.
- Evidence and Patterns workflows for incident review and historical scene
  intelligence.

## Supported Pilot Path

- Linux master/control plane is the preferred production direction.
- macOS master/control plane is supported for portable lab and demo pilots.
- Jetson Orin Nano Super 8 GB is the hardened packaged edge target.
- TensorRT engines must be built on the Jetson target or an identical Jetson
  stack.
- Helm charts render for posture validation; production k3s/Helm exercise
  remains a separate smoke before Kubernetes rollout claims.

## Install And Upgrade Notes

- Use `bin/vezor` as the product installer front door.
- Use `/first-run` to create the first tenant and tenant admin.
- Use `/platform-bootstrap` to create the first platform superadmin.
- Use **Users** for tenant and tenant-admin management after bootstrap.
- Use **Models** for source registration, TensorRT artifacts, and edge sync.
- Use **Deployment** to pair Jetson nodes and rotate node credentials.
- Use **Operations** to validate scene readiness before declaring a smoke pass.

## Security Notes

- Do not commit or share raw RTSP credentials, bearer tokens, bootstrap tokens,
  node credentials, reflector secrets, sudo passwords, or Keycloak admin
  secrets.
- Support bundles redact common credential patterns, but operators must review
  bundles before sharing.
- Platform superadmin is a high-trust role with visibility across all tenants.
  Limit it to platform operators.

## Known Limits

- Signed production packages and pinned production image manifests are still
  hardening work beyond the pilot path.
- TLS/OIDC production configuration must be completed for internet-facing
  deployments.
- Backup and restore procedures require environment-specific validation before
  production use.
- DeepStream acceleration is deferred; current Jetson path uses the packaged
  Python/TensorRT worker stack.
- STAMP/TWAMP/provider SD-WAN responder modes are future Core Link modes.
- Throughput tests are manual operator actions, not continuous interval jobs.
- Helm/k3s deployment posture should be smoke-tested separately from the
  Docker/systemd pilot path.

## Validation Rule

A whole-product smoke report must distinguish `PASS`, `FAIL`, `BLOCKED`, and
`NOT RUN`. Do not mark missing Jetson access, missing RTSP reachability, missing
model files, missing reflector secret access, missing billing usage, missing
deterministic evidence, or missing fresh-stack proof as a pass.
